# 图像识别素材规范

本目录存放用于图像识别的模板图片。脚本通过 OpenCV 模板匹配来定位屏幕上的 UI 元素。

## 命名规范

所有素材文件名使用 **英文小写 + 下划线** 命名，支持 `.png` / `.jpg` / `.jpeg` 格式。

## 所需素材清单

### 钓鱼流程相关

| 文件名 | 用途 | 截图建议 |
|--------|------|----------|
| `cast_prompt.png` | 抛竿提示 | 截取抛竿提示按钮或图标（如"按A抛竿"的文字/图标） |
| `bite_indicator.png` | 鱼上钩指示 | 截取鱼咬钩时的水花特效或"！"提示 |
| `progress_bar_bg.png` | 遛鱼进度条背景 | 截取完整的进度条背景（不含浮标和绿色区域） |
| `float_marker.png` | 黄色浮标 | 仅截取浮标本身，尽量小且清晰 |
| `green_zone.png` | 绿色安全区域 | 仅截取绿色区域部分 |
| `catch_success.png` | 钓鱼成功 | 截取"钓到了！"或鱼获界面的特征元素 |

### 商店/鱼饵相关

| 文件名 | 用途 | 截图建议 |
|--------|------|----------|
| `bait_low_warning.png` | 鱼饵不足警告 | 截取鱼饵不足时的弹窗或提示文字 |
| `shop_title.png` | 商店标题 | 截取商店界面的标题文字（如"渔获商店"） |
| `sell_confirm.png` | 出售确认 | 截取出售确认按钮或界面的特征元素 |

## 截图要求

1. **分辨率**：截图分辨率需与实际游戏画面一致，支持范围 800×600 ~ 3840×2160
2. **UI缩放**：确保游戏 UI 缩放与截图时相同
3. **裁剪精确**：尽量仅截取目标元素，减少无关背景
4. **特征唯一**：选择在屏幕上唯一的元素，避免误匹配
5. **格式**：建议使用 PNG 无损格式
6. **进程锁**：脚本仅捕获 HTGame.exe 进程画面，确保截图来自该进程窗口

## 调整匹配阈值

如果识别不准确，可以在 GUI 的「基本设置」中调整「模板匹配阈值」：
- 提高阈值（如 0.85~0.9）：更严格，减少误判但可能漏判
- 降低阈值（如 0.7~0.75）：更宽松，增加识别率但可能误判
